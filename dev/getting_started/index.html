Getting started

This page walks through installing the package, computing one Borel sum, and pointing you at the next read.

Install

Resurgence.jl is not yet registered in Julia's General registry, so install from GitHub:

using Pkg
Pkg.add(url = "https://github.com/schenkse/Resurgence.jl")

For development, clone and Pkg.activate:

git clone https://github.com/schenkse/Resurgence.jl
cd Resurgence.jl
using Pkg; Pkg.activate("."); Pkg.instantiate()
using Resurgence

Requirements

Pkg resolves these automatically — you don't need to install them by hand — but here is the full compatibility floor declared in Project.toml:

Versions follow Julia's compat semantics — "0.4" means ≥ 0.4, < 0.5; "4" means ≥ 4, < 5. Test-only extras (Aqua, Test) are not part of the runtime requirements.

A first end-to-end run

The Stieltjes/Euler series S(z) = Σ (−1)ᵏ k! zᵏ is the canonical divergent-but-Borel-summable example. At z = 1 it sums to e · E₁(1) ≈ 0.5963473623. Here's the full pipeline:

using Resurgence

# 25 coefficients in Float64. Use factorial(big(k)) so we don't overflow at k = 21.
a = Float64[(-1.0)^k * Float64(factorial(big(k))) for k in 0:24]

# Direct partial sum — diverges; the more terms you add, the worse it gets.
sum(a)
5.956716121032509e23
# Optimal truncation: stop at the smallest term. Superasymptotic accuracy.
Nstar, partial_opt, εN = optimal_truncation(a)
(Nstar, partial_opt, εN)
(1, 1.0, 1.0)
# Borel–Padé: the standard tool for factorially divergent, alternating series.
borel_pade(a; n = 10, m = 10, x = 1)
0.5963473623231734

The reference value is 0.5963473623…, so Borel–Padé already gives ~8 digits from 25 input coefficients.

Higher precision

Every method propagates the input element type. Hand it BigFloat and the intermediate Padé fit and Laplace integral are BigFloat too.

ab = BigFloat[(-1)^k * factorial(big(k)) for k in 0:40]
borel_pade(ab; n = 20, m = 20, x = BigFloat(1))
0.5963473623231727717083279835203057083988586044419772524782036271767690615350019
QuadGK and BigFloat

quadgk evaluates integration nodes in Float64 by default, so even with BigFloat input the quadrature error sits around 1e-14 unless you ask for tighter tolerances. Pass rtol = BigFloat("1e-30") (and a matching atol) explicitly to push further:

borel_pade(ab; n = 20, m = 20, x = BigFloat(1),
           rtol = BigFloat("1e-30"), atol = BigFloat("1e-30"))

Functional vs. unified API

Both of these compute the same number:

borel_pade(a; n = 10, m = 10, x = 1)
0.5963473623231734
resum(BorelPade(10, 10), a)
0.5963473623231734

Use whichever fits your code. Per-method functions are canonical; the resum(::AbstractResummation, …) layer is a thin dispatch shim that lets callers swap methods by changing one struct.
